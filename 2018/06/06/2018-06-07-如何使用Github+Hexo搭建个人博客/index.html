如何使用 Github + Hexo 搭建个人博客（2026 图文版）

发表于 2018-06-07 更新于 2026-07-03 阅读次数： 本文字数： 2.2k 阅读时长 ≈ 8 分钟

前言

两年前，我写下了这个博客的第一篇文章，记录了当时搭建 Hexo 博客的过程。时光荏苒，Hexo 和 GitHub Pages 都经历了多次版本更新，原来的教程已经有些过时。最近我重新部署了这个博客，踩了不少坑，也学到了很多新东西。这篇文章将以最新的 Hexo 8.x + NexT 8.x 为主线，带你从零开始搭建一个属于自己的博客。

你将获得：一个完全免费、高度可定制、支持 Markdown 写作的个人博客，自带访客统计、本地搜索、代码高亮等功能。

第一步：创建 GitHub 仓库

GitHub Pages 的规则是：仓库名必须为 {用户名}.github.io。例如我的 GitHub 用户名为 694587769，仓库名就是 694587769.github.io。

  1. 登录 GitHub，点击右上角 + → New repository
  2. Repository name 填写 你的用户名.github.io
  3. 选择 Public（公开仓库）
  4. 勾选 Add a README file（方便后续操作）
  5. 点击 Create repository

注意：仓库名必须是 用户名.github.io，大小写敏感。如果用户名有大写字母，仓库名中请使用小写。

创建完成后，你会得到一个空仓库，地址类似：https://github.com/你的用户名/你的用户名.github.io

7.4 配置 GitHub Token（推荐）

GitHub 已经不再支持密码认证，推荐使用 Personal Access Token：

  1. 进入 GitHub → Settings → Developer settings → Personal access tokens → Tokens (classic)
  2. 点击 Generate new token (classic)
  3. 勾选 repo 权限
  4. 生成后复制 Token（只显示一次！）

常见问题排查

问题一：页面显示 404

  • 检查仓库名是否正确（用户名.github.io，大小写敏感）
  • 确认 GitHub Pages 设置中 Source 选择的是 master 分支

问题二：样式丢失 / 资源加载失败

  • 检查 _config.yml 中 url 和 root 配置：

问题三：部署后页面没更新

  • 可能是浏览器缓存，按 Ctrl+Shift+R 强制刷新
  • GitHub Pages 构建需要时间，可以查看 Actions 页面确认构建状态

问题四：不蒜子统计不显示

  • 检查 themes/next/_config.yml 中 busuanzi_count.enable 是否为 true
  • 检查浏览器控制台是否有 CDN 加载错误
  • 不蒜子服务偶尔会不稳定，稍等几分钟再试

总结

至此，你已经拥有了一个功能完善的个人博客：

  • Hexo 框架：快速生成静态页面
  • NexT.Mist 主题：简洁优雅的视觉风格
  • GitHub Pages：免费、稳定的托管服务
  • 不蒜子统计：实时访客计数
  • 本地搜索：无需后端即可全文搜索
  • 代码高亮：支持多种编程语言

整个博客从零到上线，大约需要 30 分钟。后续只需要用 Markdown 写文章，一条命令就能部署到线上。

两年前我第一次搭建这个博客时，用的是 Hexo 3.x 和 NexT 5.x。如今升级到 Hexo 8.x + NexT 8.x，配置更加简洁，功能也更丰富了。希望这篇教程能帮你少走一些弯路，享受写作的乐趣。

博客的意义不在于技术有多复杂，而在于持续记录与分享。愿你在写作中找到属于自己的声音。
