# 三国·幸存者 游戏规格文档

## 1. 产品概述

**项目名称**: 三国·幸存者 (Three Kingdoms Survivors)

**项目概述**: 一款以中国三国时代为背景的吸血鬼幸存者类游戏，玩家扮演蜀国武将，在无尽的敌军包围中生存、升级、获取装备，体验紧张刺激的割草快感。

**核心价值**:
- 融合三国历史人物与东方玄幻元素
- 简单易上手的操作，深度的技能搭配策略
- AI生成的视觉素材打造独特的东方美学风格
- 流畅的即时战斗体验，支持长时间游玩

## 2. 核心功能

### 2.1 游戏模式

| 模式 | 描述 |
|------|------|
| 经典模式 | 标准幸存者模式，时间无限，难度递增 |
| 挑战模式 | 特定限制条件下挑战高分 |

### 2.2 角色系统

初始可玩角色：赵云、关羽、张飞（各具独特被动技能）

| 角色 | 初始武器 | 被动技能 | 特点 |
|------|----------|----------|------|
| 赵云 | 青虹剑 | 龙胆之心 | 移动速度+15% |
| 关羽 | 青龙偃月刀 | 武圣之魂 | 暴击率+20% |
| 张飞 | 丈八蛇矛 | 燕人怒吼 | 经验获取+25% |

### 2.3 武器系统

| 武器类型 | 名称 | 效果描述 |
|----------|------|----------|
| 近战 | 青虹剑 | 360°旋转攻击周围的敌人 |
| 近战 | 青龙偃月刀 | 前方扇形范围高伤害攻击 |
| 远程 | 诸葛连弩 | 自动追踪发射箭矢 |
| 远程 | 火计 | 落下火焰造成范围伤害 |
| 辅助 | 八卦阵 | 角色周围形成护盾 |
| 辅助 | 医疗兵法 | 定期恢复生命值 |

### 2.4 升级系统

- 每级提供3个随机技能选择
- 技能可升级强化效果
- 武器等级最高5级

### 2.5 敌人系统

三国魏、蜀、吴三方士兵与将领作为敌人出现：

| 敌人类型 | 血量 | 伤害 | 特点 |
|----------|------|------|------|
| 魏国步兵 | 低 | 低 | 数量多 |
| 魏国弓兵 | 中 | 中 | 远程攻击 |
| 吴国水兵 | 中 | 中 | 中距离 |
| 吕布 | 极高 | 极高 | BOSS级敌人 |

### 2.6 资源与经济

| 资源类型 | 用途 |
|----------|------|
| 经验值 | 升级获取新技能 |
| 金币 | 局内无用途（为未来皮肤系统预留） |
| 游戏时间 | 核心挑战指标 |

## 3. 核心流程

### 3.1 游戏主流程

```
开始界面 → 选择角色 → 游戏准备 → 生存战斗 → 角色死亡 → 结算展示 → 返回开始界面
```

### 3.2 战斗流程

```
角色移动（自动攻击）→ 击杀敌人获得经验 → 升级选择技能 → 敌人难度递增 → 生存至最后
```

### 3.3 技能选择流程

```
获得足够经验 → 升级暂停 → 显示3个随机选项 → 玩家选择 → 继续战斗
```

## 4. 用户界面设计

### 4.1 设计风格

**整体美学**: 东方水墨风格与现代游戏元素的融合

**配色方案**:
- 主色调: `#1a1a2e` (深墨蓝) - 战场背景
- 次要色: `#16213e` (深靛蓝) - UI面板
- 强调色: `#e94560` (战血红) - 敌人、危险
- 辅助色: `#f4d35e` (金色) - 经验、奖励
- 角色色: `#4ecca3` (青绿) - 玩家角色、技能

**字体选择**:
- 标题: "ZCOOL XiaoWei" (站酷小薇体) - 中文毛笔风格
- 正文: "Noto Sans SC" - 清晰易读

**UI风格**:
- 圆角卡片式设计
- 半透明毛玻璃效果
- 水墨纹理装饰

### 4.2 页面结构

| 页面 | 模块 | 元素 |
|------|------|------|
| 开始界面 | 标题区 | 游戏LOGO、背景图 |
| | 操作区 | 开始游戏按钮、角色选择 |
| | 信息区 | 最高记录显示 |
| 游戏界面 | 顶部状态栏 | 存活时间、生命值、等级 |
| | 主游戏区 | 角色、敌人、特效 |
| | 底部技能栏 | 当前武器、已装备技能图标 |
| 升级界面 | 暂停遮罩 | 全屏暗化 |
| | 选择卡片 | 3个技能选项横向排列 |
| 结算界面 | 结果展示 | 存活时间、击杀数、等级 |
| | 操作区 | 重新开始、主菜单按钮 |

### 4.3 响应式设计

- 默认桌面端 1920x1080
- 自适应缩放保持比例
- 触摸设备支持虚拟摇杆

## 5. 视觉效果规格

### 5.1 AI生成素材规格

| 素材类型 | 尺寸 | 格式 | 用途 |
|----------|------|------|------|
| 角色立绘 | 512x512 | PNG | 游戏内角色 |
| 敌人精灵 | 256x256 | PNG | 各类型敌人 |
| 武器图标 | 128x128 | PNG | 技能栏显示 |
| 背景图 | 1920x1080 | JPG | 游戏背景 |
| UI装饰 | 可变 | PNG/SVG | 边框、图标 |

### 5.2 动画效果

- 角色移动: 8方向行走动画
- 攻击特效: 武器对应特效（剑气、火焰等）
- 敌人死亡: 消散/击退动画
- UI过渡: 淡入淡出、缩放弹跳

## 6. 技术架构

### 6.1 技术栈

| 类别 | 技术 | 说明 |
|------|------|------|
| 渲染 | HTML5 Canvas | 2D游戏渲染 |
| 逻辑 | Vanilla JavaScript | 原生JS保证性能 |
| 构建 | 单HTML文件 | 无需构建工具 |
| 素材 | AI生成图片 | Base64内嵌 |

### 6.2 游戏架构

```
GameEngine
├── InputManager (输入处理)
├── EntityManager (实体管理)
│   ├── Player (玩家角色)
│   ├── Enemy[] (敌人池)
│   └── Projectile[] (投射物)
├── CollisionSystem (碰撞检测)
├── SkillSystem (技能系统)
├── UIManager (界面管理)
└── Renderer (渲染器)
```

### 6.3 性能目标

- 目标帧率: 60 FPS
- 最大同屏敌人: 200+
- 内存占用: < 200MB
